Página de Ayuda para la Herramienta VBA de Análisis Clúster.

Análisis cluster es una aplicación informática desarrollada en VBA 6.0 y VBA 7.1 para la asignatura de Herramientas de Cálculo Actuarial. La aplicación tiene finalidad educativa en el ámbito académico de la UCIII de Madrid. Se utiliza para realizar una segmentación y división de datos en base a sus características homogéneas y heterogéneas, para lo cual aplica funciones de clusterización propias del análisis multivariante. La aplicación se ejecuta en entorno MS Excel y cuenta con tres partes diferenciadas en su funcionamiento: Un módulo de lectura de datos, un módulo de cálculo y un módulo de resultados. El acceso a la interfaz de la aplicación puede realizarse de dos modos. Mediante el botón “formulario” situado en la hoja “pruebas” o mediante el menú/máster CAF/ Cluster 1.1.

La aplicación cuenta con un único formulario desde el cual pueden realizarse todas las opciones de segmentación desarrolladas. Una vez abierto el formulario en primer lugar es necesario posicionarse sobre el control “refedit” para acceder a la lectura de los datos. Una vez minimizada la ventana de formulario se selecciona el rango deseado y se vuelve a presionar el control de “refedit” o enter. La aplicación tiene preparado el rango para trabajar.

A continuación debe seleccionarse algunos parámetros básicos de la estructura de los datos, como la posibilidad de que existan nombres en los la cabecera de las columnas (variables) y/o nombres en la primera columna (identificadores) El módulo de lectura de datos permite leer un rango de información en formato Excel (xls,xlsx) y transformarlo a una matriz de cálculo en formato Array. De este modo se aceleran los procesos de cálculo al prescindir del objeto rango. Permite, además, realizar una selección de las variables a utilizar en el proceso siguiente redimensionando adecuadamente las matrices. Para cargar los datos al formulario y entrar en el frame de selección de variables es necesario pulsar el botón “cargar datos”. En caso de no existir nombres en las cabeceras la aplicación genera un identificador de la variable.
Una vez pasadas las variables al listbox, se puede seleccionar aquellas variables de interés por el usuario, o bien desactivar aquellas que no se traten de datos numéricos cuantitativos. El frame incluye un selector de todas / ninguna variable. Tras la selección final, mediante el botón “Generar matriz” los datos se almacenan en la matriz gobal de datos para continuar el proceso de segmentación.
En caso de que existan celdas vacías o con datos tipo “string” se obtendrán mensajes de aviso. El módulo de análisis facilita la elección de diferentes opciones de segmentación, incluidos establecer un número determinado de grupos iniciales. Los análisis incluyen varios tipos de criterios de agrupación basados en diferentes medidas de distancias, submétodos de agrupación y dos estrategias de segmentación (similaridad y distancia).

Las opciones de distancias que incluye la aplicación:

  • Distancia euclídea
  • Distancia de Mahalanobis
  • Distancia de Manhattan
  • Distancia de Chebyshev
  • Distancias de Makowski Para estas distancias pueden elegirse los siguiente métodos:
  • Single linkage
  • Complete linkage
  • WPGMA
  • UPGMA La aplicación incluye un checkbos para lanzar el gráfico de dendrograma. Este gráfico se conserva en la misma ubicación que el archivo de datos hasta que se cierra la apliación mediante el botón “cerrar”. Paralelamente puede elegirse el número de grupos que se quiere crear de forma supervisada y la opción de cambiar el valor del exponente (p) para el caso de la distancia de Minkowski.

El módulo de salida se compone de un comando para crear una hoja de resultados nueva.

En caso de elegirse la opción “dendrograma” mediante el checkbox, la apliacón lanzará una nueva ventana en al cual se ha cargado una imagen del dendrograma, que incluye:

  • Línea de corte según los grupos elegidos
  • Coeficiente cofenético (-1,1) para determinar la calidad de la segmentación.

El módulo de salida ofrecerá un reporte de los grupos creados y diferentes gráficos dinámicos que permitan interpretar los resultados. Según el tipo de segmentación elegida (Jerárquico o no Jerárquico el tipo de gráfico será diferente.

Respecto a la clarificación no jerárquica, la aplicación incluye el algoritmo K-medias. En este caso las opciones del algoritmo son:

  • Elección de la matriz original para realizar los cálculos
  • Elección de matriz de corelaciones o covarianas

En este caso incluye además las opciones de Convergencia y número de ciclos de cálculo. En este algoritmo se han implementado funciones que mejoran el rendimiento notablemente de los cálculos matriciales.

El gráfico generado mediante k-medias es una imagen cargada en una ventana emergente. Su código incluye la creación de una tabla dinámica.
